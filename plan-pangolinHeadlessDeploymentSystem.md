# Plan: Pangolin Headless Deployment System

## Overview
Create a two-part system for headless Pangolin deployment that functionally recreates the Pangolin installer (https://github.com/fosrl/pangolin/tree/main/install/main.go), except that instead of directly installing Pangolin, it outputs a docker-compose file that will install it as though the script had been run locally, and then run it.

The two parts are:
1. **Deployer Docker Image** - Auto-generates config files from environment variables (what the installer does at runtime)
2. **Interactive GitHub page** - Collects user input and generates the docker-compose file (what the installer prompts for)

## Files Needed for Pangolin Deployment

### 1. docker-compose.yml
- pangolin service (port 3001, 3002)
- gerbil service (ports 51820, 21820, 80, 443)
- traefik service (shares network with gerbil)
- **NEW: deployer service** (runs first, creates configs)

### 2. config/config.yml
- gerbil settings (start_port, base_endpoint, subnet_group, block_size)
- app settings (dashboard_url, log_level, telemetry)
- domains configuration (base_domain, cert_resolver)
- server settings (secret, cors)
- Optional: email, postgres, rate_limits, flags, orgs, traefik

### 3. config/traefik/traefik_config.yml
- API settings, providers, certificates resolver, entry points

### 4. config/traefik/dynamic_config.yml
- Middlewares, routers, services

## Environment Variables to Support

### Required:
- DOMAIN (base domain, e.g., example.com)
- DASHBOARD_URL (full URL, e.g., https://pangolin.example.com)
- LETSENCRYPT_EMAIL (for SSL certificates)

### Edition:
- We only support Community Edition

### Optional with defaults:
- GERBIL_START_PORT (default: 51820)
- GERBIL_BASE_ENDPOINT (defaults to pangolin.{DOMAIN})
- LOG_LEVEL (default: info)
- ORG_BLOCK_SIZE (default: 24)
- ORG_SUBNET_GROUP (default: 100.90.128.0/20)
- ENABLE_IPV6 (default: true)
- ENABLE_GEOBLOCKING (default: false) - downloads MaxMind database
- INSTALL_GERBIL (default: true) - tunneled connections support
- CONTAINER_RUNTIME (default: docker) - docker or podman

### Sensitive (can be blank, user fills later):
- SERVER_SECRET (auto-generated if not provided)
- EMAIL_SMTP_PASS
- POSTGRES_CONNECTION_STRING

### Email configuration (optional):
- EMAIL_ENABLED (default: false)
- EMAIL_SMTP_HOST
- EMAIL_SMTP_PORT (default: 587)
- EMAIL_SMTP_USER
- EMAIL_NO_REPLY

### Certificate files:
- SSL_CERTIFICATE (fullchain.pem - base64 encoded)
- SSL_PRIVATE_KEY (privkey.pem - base64 encoded)
- Users can provide certificate files OR skip and fill in manually later
- GitHub page will encode files to base64 for the deployer to decode

## Implementation Steps

### Phase 1: Deployer Docker Image
1. Create Dockerfile based on alpine/curl or similar
2. Create entrypoint script that:
   - Reads all PANGOLIN_* env vars
   - Generates config/config.yml from templates
   - Generates traefik config files
   - Decodes base64-encoded certificate files and writes to config/letsencrypt/
   - Creates directory structure
   - Exits 0 when done (other containers mount same volumes)

### Phase 2: GitHub Page Generator
1. Create single HTML file with:
   - Form collecting all required config
   - Option to upload/paste certificates or skip
   - Preview of generated docker-compose.yml
   - Download buttons (monolithic or split with .env file)

### Phase 3: Deployer Container Logic
1. Deployer runs as init container
2. Creates all config directories
3. Writes all config files from env vars
4. Handles certificate files
5. Exits 0 when done (other containers mount same volume)
6. Other containers depend on deployer with `condition: service_completed`

## Service Dependency Chain

Based on [Pangolin Docker Compose](https://docs.pangolin.net/self-host/manual/docker-compose):

```
deployer (runs first, exits)
    ↓ condition: service_completed
pangolin (healthy)
    ├─→ gerbil (depends_on pangolin healthy)
    └─→ traefik (depends_on pangolin healthy, network_mode: service:gerbil)
```

Key points:
- Deployer runs first and completes (exits) before others start
- pangolin must be healthy before gerbil/traefik start
- traefik uses `network_mode: service:gerbil` to share gerbil's network
- Config files are generated by deployer into mounted volume, not manually created

## Output Formats

### Option 1: Monolithic docker-compose.yml
- All env vars inline in environment: section
- No separate .env file needed

### Option 2: Split format
- docker-compose.yml with env_file: reference
- separate .env file with all variables

## Certificate Handling
- User can provide base64-encoded certs in env vars
- Or skip and leave blank (noted in output)
- Deployer will write placeholder or actual certs

## Verification
1. Deployer image builds successfully
2. Config files are generated correctly
3. GitHub page generates valid docker-compose.yml
4. Both output formats work with docker compose
